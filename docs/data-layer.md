# Data Layer

## Stack

- **Database**: PostgreSQL 17 + pgvector extension
- **ORM**: Drizzle ORM 0.45
- **Driver**: `postgres` (porsager/postgres)
- **Migrations**: Drizzle Kit → `drizzle/` directory

## Schema Location

Schema defined in `packages/orchestrator/src/db/schema.ts`. Query helpers in `queries.ts`. Connection in `drizzle.ts`.

## Tables

| Table | Purpose | Key columns |
|-------|---------|-------------|
| `agent_definitions` | Agent templates | name (unique), mission, model, default_budget, capabilities |
| `agents` | Running agent instances | definition_id (FK), status, sandbox_id, budget_total/spent, heartbeat_at |
| `agent_memories` | Per-agent memory entries | agent_id (FK), content, embedding (vector), importance_score, memory_type |
| `agent_sessions` | Agent execution sessions | agent_id (FK), status, started_at, ended_at, summary |
| `checkpoints` | Agent progress snapshots | agent_id (FK), session_id (FK), current_goal, progress_state, handoff_prompt |
| `shared_knowledge` | Cross-agent knowledge base | content, embedding (vector), entry_type, category, importance_score |
| `skills` | Learned agent behaviors | name (unique), version, trigger_condition, steps, status, success/failure counts |
| `audit_log` | Security and action audit trail | agent_id (FK), action_type, outcome, security_event |
| `financial_transactions` | Budget tracking | agent_id (FK), type, amount, category, pre_auth_verified |
| `llm_usage_logs` | LLM cost tracking | agent_id (FK), model_provider, model_id, input/output_tokens, computed_cost_usd |
| `identities` | Agent identity credentials | agent_id (FK), identity_type, provider, identifier, status |
| `secrets` | Encrypted secret storage | name (unique), secret_type, encrypted_value (bytea), allowed_agents, allowed_domains |
| `pipelines` | Multi-stage workflows | name, goal, stages (jsonb), current_stage, status, budget |
| `alert_rules` | Monitoring alert definitions | name, condition (jsonb), severity, enabled, notify_discord |
| `alert_events` | Triggered alert instances | rule_id (FK), agent_id (FK), severity, message, acknowledged |
| `operator_credentials` | WebAuthn passkeys | credential_id (bytea), public_key (bytea), counter |
| `system_config` | Global system settings | Singleton (id=1), heartbeat_interval, revenue splits, active_hours |

## Custom Types

```typescript
// pgvector — 1536 dimensions for embeddings
const vector = customType<{ data: number[]; driverData: string }>({
  dataType() { return "vector(1536)"; },
  toDriver(value) { return `[${value.join(",")}]`; },
  fromDriver(value) { return value.slice(1, -1).split(",").map(Number); },
});

// bytea — binary data (WebAuthn keys, encrypted secrets)
const bytea = customType<{ data: Buffer; driverData: Buffer }>({
  dataType() { return "bytea"; },
});
```

## Primary Keys

All tables use `uuid("id").primaryKey().defaultRandom()` except `system_config` which uses `integer("id").primaryKey().default(1)` (singleton).

## Indexing Strategy

- Foreign keys are indexed (`agents_definition_id_idx`, `agent_memories_agent_id_idx`, etc.)
- Status columns indexed for filtered queries (`agents_status_idx`, `pipelines_status_idx`)
- Timestamp columns indexed for range queries (`audit_log_timestamp_idx`, `llm_usage_created_idx`)
- Compound indexes for common query patterns (`agent_memories: agent_id + importance_score`)
- Security events indexed separately (`audit_log_security_idx`)

## Relations

Defined via `relations()` from `drizzle-orm`:

```text
agentDefinitions ──1:N──▶ agents
agents ──1:N──▶ agentMemories, agentSessions, checkpoints, identities
agents ──self──▶ agents (parent/child hierarchy)
agentSessions ──1:N──▶ checkpoints
alertRules ──1:N──▶ alertEvents
```

## Query Patterns

Use helpers in `queries.ts`:

```typescript
import { findAgentById, insertAgent, updateAgent } from "./db/queries.js";

const agent = await findAgentById(db, agentId);
const newAgent = await insertAgent(db, { definitionId, name, ... });
const updated = await updateAgent(db, id, { status: "running" });
```

`updateAgent` and `updateDefinition` automatically set `updatedAt: new Date()`.

For queries not covered by helpers, use Drizzle's query builder directly:

```typescript
import { eq } from "drizzle-orm";
import * as schema from "./db/schema.js";

const memories = await db
  .select()
  .from(schema.agentMemories)
  .where(eq(schema.agentMemories.agentId, agentId));
```

## Migrations

```bash
pnpm db:migrate       # Apply pending migrations
pnpm db:studio        # Open Drizzle Studio (TTY required)
```

- Migrations live in `drizzle/`.
- Generated by Drizzle Kit from schema changes.
- Never edit existing migration files.
- Test migrations against Testcontainers PostgreSQL before applying to prod.
